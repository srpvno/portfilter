I made this port filter for personal use, as a way to study the common port numbers as well as keep track of any important ones when it comes to the basics of networking fundamentals. Also, it's a useful tool to quickly search between port numbers and the service names associated with it.





Port Number Transport Protocol Service Name Description
20 TCP File Transfer Protocol - Data (FTP) Once port 21 has established the connection, this port is used to transfer data files across the ports.
21 TCP File Transfer Protocol - Control/Command (FTP) Establishes the connection as well as user authentication between ports before the files can actually be sent across.
22 Both Secure Shell (SSH) SSH is a cryptographic network protocol for operating network services securely over an unsecured network. Typical applications include remote cmd login and execution, but any network can be secured with SSH.
Files leaked by Edward Snowden, however, indicate that the NSA can sometimes decrypt SSH, allowing them to read the contents of SSH sessions.
23 TCP Telnet Telnet services essentially offer to accept remote console terminal connection from any client device across the Internet. This makes Telnet quite powerful and, without proper security, a significant security concern.
Due to the tremendous potential for abuse, hackers generally take an immediate interest in any system that is presenting an open Telnet port to the Internet. Make sure to shut down any open Telnet port on your system!
25 TCP Simple Mail Transfer Protocol (SMTP) SMTP is the protocol used to shuttle email across the Internet from one mail server to another. SMTP servers open and listen for incoming connection on port 25. Another SMTP server, or a personal email client, will connect to the server on its port 25 to transfer some email into it for subsequent forwarding toward its destination.
50, 51 # IPSec This is used mainly over VPNs. It authenticates and encrypts the packets of data sent over an IP network.
53 Both Domain Name Server (DNS) DNS is the "glue" that translates human-readable domain names into their IP address equivalents. Incoming UDP packets carry queries which expect a short reply and TCP connections carry queries requiring longer and more complete replies.
67, 68 UDP Dynamic Host Configuration Protocol (DHCP) This is how your computer gets it unique IP address. When a system starts up on a network, it must first request an IP address (assuming it is not using a static IP address), and it does this by the broadcast request.
Ports 67 and 68 are used in order to specify the request (68) and tell the server (67) to reply back.
69 UDP Trivial File Transfer Protocol (TFTP) TFTP is a simple protocol that allows a client to place or grab a file form a remote host. It is used to transfer firmware images and configuration files to network appliances like routers, firewalls, IP phones, etc. It's the protocol of choice for the initial stages of any network booting strategy like BOOTP, PXE, BSDP, etc.
80 TCP HyperText Transfer Protocol (HTTP) The default port number for a Web server. Experimental services may sometimes be run at port 8080.
110 TCP Post Office Protocol (POP3) A client pulls emails from the email server. By doing so, a local file is stored [PST] on the local device. The email server has now removed specified email from its server and thus, is only present on the local device.
119 TCP Networks News Transport Protocol (NNTP) Port 119 hosts the servers of the famous and infamous Internet USENET newsgroup world. NNTP servers push and pull news articles to and from other NNTP servers over port 119, and news reading (and writing) clients talking to news servers over the same port.
123 UDP Network Time Protocol (NTP) The NTP is a clean, simple, lightweight and efficient protocol allowing clients to query servers for the current time (including the date), thus, allowing every client in various timezones to sync properly.
137-139 Both NetBIOS In the name of backward compatibility, Windows 2000 and later OS's continued to suppor tthe original NetBIOS port trio. However, they've now moved their NetBIOS services over to port 445 - but have now created even more serious security problems with that port.
143 Both Internet Message Access Protocol (IMAP4) A client pulls an email from the email server and creates a local copy unto the local device. Any changes on the local device will sync with the email server. The email server saves a copy unto the server itself. Thus, if the client decides to switch devices, it can pull the any changed data from the email server once again.
161, 162 Both Simple Network Management Protocol (SNMP) SNMP is for collecting and organizing information about managed devices on IP networks and for modifying that information to change device behavior. Devices that typically support SNMP include cable modems, routers, switches, servers, workstations, printer, etc.
389 Both Lightweight Directory Access Protocol (LDAP) LDAP is a modern and popular Internet directory access protocol used by many systems and services. Most Windows users will encounter it due to the port running while it is open.
443 Both HTTP with SSL (HTTPS) This port is used for secure web browser communication. Data transferred across such connections are highly resistant to eavesdropping and interception. Moreover, the identity of the remotely connected server can be verified with significant confidence. Web servers offering to accept and establish secure connections listen on this port for connections from web browsers desiring strong communication security.
3389 TCP Remote Desktop Protocol (RDP) This port is used by Microsoft's "Terminal Server" which was renamed to Remote Desktop for their appearnace in Windows XP.
